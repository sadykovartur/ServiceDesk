# Lab 05 — Формы и валидация: Create/Edit ticket + согласованные ошибки

## Цель
Сделать формы создания/редактирования тикета, валидацию на фронте и бэке.

## Что сделать
Frontend:
1) Форма Create Ticket:
   - React Hook Form + Zod (рекомендуется)
   - отображение ошибок под полями
2) UX:
   - disable submit в процессе
   - Snackbar/Alert об успехе/ошибке

Backend:
3) Валидация входных DTO:
   - обязательные поля
   - длины строк
4) Возврат ошибок в едином формате (ProblemDetails).

## Критерии приёмки
- валидация работает на клиенте и сервере
- ошибки сервера отображаются в UI
- тикет реально создаётся в БД и виден в списке

## Команды проверки
- npm run build
- dotnet build
- docker compose up
