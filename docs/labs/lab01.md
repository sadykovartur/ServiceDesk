# Lab 01 — Спецификация в Figma (flows + экраны + состояния)

## Цель
Создать дизайн-спецификацию (Figma), которая будет источником требований для UI и API.

## Что сделать
1) Определить роли и основные сценарии (минимум 2):
   - Student: создать заявку -> посмотреть в списке -> открыть карточку -> написать комментарий
   - Operator: открыть очередь -> назначить на себя -> сменить статус -> ответить комментарием

2) Сделать user flow (FigJam или Figma).
3) Сделать wireframes (6–10 экранов).
4) Для 2–3 ключевых экранов добавить состояния:
   - loading, empty, error, validation
5) (Опционально) Сделать hi-fi варианты, используя MUI/Material стиль.

## Набор экранов (минимум)
- Login
- Tickets List (Student)
- Create Ticket
- Ticket Details (comments + actions)
- Queue (Operator)
- Admin Categories (если роль Admin реализуете)

## Критерии приёмки
- есть ссылка на прототип (Prototype link) и доступ на просмотр
- есть минимум 6 экранов
- есть минимум 2 user flow
- есть состояния loading/empty/error/validation хотя бы для 2 экранов

## Сдача
- ссылка на Figma (в README или в LMS)
- краткий список сценариев (bullet list)
